# 📦 Archive des fichiers de debug

Ce dossier contient tous les scripts et fichiers markdown de debug/test créés pendant le développement.

## 🗂️ Contenu

### Scripts de test Python
- `check_*.py` - Scripts de vérification (extraction PDF, metadata, etc.)
- `test_*.py` - Scripts de test (gpt-oss, pymupdf, rag, etc.)
- `search_*.py` - Scripts de recherche dans les PDFs
- `extract_pages.py` - Script d'extraction de pages

### Fichiers markdown de debug
- `DEBUG_*.md` - Diagnostics de problèmes
- `DIAGNOSTIC_*.md` - Analyses de dysfonctionnements
- `SOLUTION_*.md` - Solutions aux problèmes rencontrés
- `JOURNAL_DEBUG_SESSION.md` - Journal de debug
- `INSTALLER_PYTORCH_CUDA.md` - Notes d'installation
- `ETAPES_FINALES.md` - Étapes de finalisation
- `PROBLEME_*.md` - Documentation de problèmes
- `NOUVELLE_FONCTIONNALITE_*.md` - Notes sur nouvelles features

### Autres
- `claude01.txt` - Logs/notes Claude

## 🔧 Utilisation

Ces fichiers sont **conservés pour référence** mais ne sont plus nécessaires au fonctionnement de l'application.

Si tu as besoin de :
- Debugger un problème similaire → Consulte les fichiers markdown
- Tester l'extraction PDF → Utilise les scripts `test_*.py`
- Vérifier la vectorDB → Utilise `check_vectordb_metadata.py`

## 🗑️ Suppression

Tu peux **supprimer ce dossier entier** sans affecter l'application si tu es sûr de ne plus en avoir besoin.
